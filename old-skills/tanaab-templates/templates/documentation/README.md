# Documentation Templates

Use these templates when standardizing repository README files and root-level template-adoption docs
under the Tanaab documentation rules.

## Available Templates

- `readme-full/README.md`: use when the repository can keep all durable user-facing documentation in one README.
- `readme-github-action/README.md`: use when the repository's primary artifact is a GitHub Action and the README must carry the action contract.
- `readme-docs-wrapper/README.md`: use when the README should stay lightweight and delegate durable docs to a VitePress docs site.
- `TEMPLATE-SETUP.md`: use in a GitHub repository template when the repo needs a root-level adoption spec for humans and AI agents.

For the detailed decision rule, section expectations, and shared `Contributors` embed standard, see [../../../tanaab-documentation/references/readme-standards.md](../../../tanaab-documentation/references/readme-standards.md).
